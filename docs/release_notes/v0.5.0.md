# v0.5.0 — ideas 命令重构：外部数据、缓存加速、init 子命令

## 主要变更

### ideas 数据外置 + 索引缓存

- **移除 embedded ideas.json**：Binary 体积减少 **~51 MB**，数据和代码彻底分离
- **`apimart-cli ideas init`** 子命令：自动从 GitHub 下载提示词数据到 `~/.config/apimart/ideas.json`
- **BM25 搜索索引缓存**（默认开启）：首次搜索自动构建并缓存到 `~/.config/apimart/ideas.index`，后续搜索跳过索引构建，速度提升 **50-300ms**
- **SHA256 一致性校验**：`ideas.json` 内容变更时自动重建缓存，无需手动干预
- 数据文件路径和缓存行为可通过 `config.yaml` 的 `ideas` 段配置

### ideas 搜索增强

- **BM25 + n-gram 混合搜索算法**：更准确的相关性排序（Title Match 加权）
- **并行 tokenization**：利用多核 CPU 加速 26K 条数据的索引构建
- **CJK 加速路径**：中日韩字符的快速分词优化
- **`--find-image` 去重修复**、JSON 损坏恢复

### 配置系统改进

- 新增 `IdeasConfig` 配置类型（data_path / index_path / cache_enabled）
- 修复 `PersistentPreRunE` 中 API Key 检查未覆盖 ideas 子命令的问题
- 全量 Config 通过 `SharedConfig.Cfg` 暴露给子命令

### 其他

- `ideas.json` 源文件从 `cmd/` 移至 `docs/`

## 升级说明

```bash
# 首次使用需要初始化数据
apimart-cli ideas init

# 原有用法不变
apimart-cli ideas "cinematic portrait"
```

缓存默认开启，无需额外配置。
