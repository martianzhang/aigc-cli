### 大文件拆分（7 个模块 → 36 个文件）
对项目最大的 7 个源文件按功能域拆分，每个文件控制在 200 行左右，便于 AI 分析和团队维护：

| 模块 | 原大小 | 拆成 |
|---|---|---|
| `image.go` | 823 行 | 5 个文件 |
| `video.go` | 901 行 | 5 个文件 |
| `midjourney.go` | 1351 行 | 7 个文件 |

拆分原则：命令定义、请求构建、策略路由、Provider 执行器、辅助函数各归其位。

## 🐛 修复

### Windows TUI 修复
- **输入框不可见**：viewport 高度从硬编码 `-12` 改为动态 `- chromeHeight`（下限 3），防止终端尺寸偏小时 viewport 占满屏幕
- **闪动错行**：流式响应内容过滤 `\r` 字符（Windows 终端遇 `\r` 光标回行首导致显示错乱）
- **边框渲染不稳定**：输入框从 `RoundedBorder` 改为仅左边 `┃` 竖条
- **输入框窄于界面**：移除 `MaxWidth=80` 硬限制，改为 999（真实宽度由 `SetWidth` 动态决定）
- **最小宽度下限**：10px 防止 `m.width` 为 0 时出现负宽度

### 中文编码修复
- **`!shell` 命令中文乱码**：Windows 下 `cmd`/`pwsh` 输出编码为 GBK/CP936，Go 按 UTF-8 解码导致乱码
  - cmd：前缀 `chcp 65001 >NUL &` 切换到 UTF-8 代码页
  - PowerShell：前缀设置 `$OutputEncoding = [Console]::OutputEncoding = UTF8`
