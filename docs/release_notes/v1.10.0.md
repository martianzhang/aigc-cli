# v1.10.0 — 本地语音合成与语音识别

本次更新引入基于 sherpa-onnx 的**本地离线 TTS 和 ASR** 管线，无需联网、无需 API Key，数据不出设备。

## 主要新功能

### 🎤 本地 TTS（文字转语音）

```bash
aigc-cli audio init                    # 下载默认模型（kokoro + sense-voice）
aigc-cli audio tts --local --input "你好"  # 语音合成
```

- **Kokoro 多语言 TTS**：53 种音色，支持中/英/日/韩/法，默认中文女声晓晓（zf_xiaoxiao）
- **VITS 系列**：vits-ljs（美式英语）、vits-vctk（英式、109 说话人）、3 个中文模型、粤语
- **Matcha-TTS**：中英双语模型
- 音色支持名字（`--voice zf_xiaoxiao`）或编号（`--voice 47`）
- `--list-voices` 查看所有可用音色

### 🎯 本地 ASR（语音转文字）

```bash
aigc-cli audio asr --local --input speech.wav  # 语音识别
```

- **SenseVoice**：中文语音识别最佳，阿里达摩院模型
- **Whisper-tiny**：多语言通用
- 支持 WAV / MP3 / FLAC / OGG 输入格式

### 📦 模型管理

```bash
aigc-cli audio init                          # 下载默认 TTS + ASR
aigc-cli audio init --list                   # 查看所有可下载模型（12 种）
aigc-cli audio init --url <url> --name x     # 从任意 URL 下载
aigc-cli audio init --list-voices --model kokoro  # 查看音色
```

### ⚙️ 配置

```yaml
defaults:
  audio:
    local: true              # 启用本地模式
    speak_model: kokoro      # 本地 TTS 模型
    transcribe_model: sense-voice  # 本地 ASR 模型
    voice: "zf_xiaoxiao"     # 默认音色
    format: wav              # 输出格式
```

## 依赖变更

- 新增 `github.com/k2-fsa/sherpa-onnx-go`（CGO，预编译库自动下载）
- 新增纯 Go 音频解码库：go-mp3、flac、oggvorbis
- CGO 要求：macOS/Linux 默认开启，Windows 需 MinGW

## 其他改进

- `onnxrt.LibPath` 按平台优先级查找动态库（修复同时存在 .dylib 和 .dll 时的加载错误）
- `service.SaveResourceWithBearer` 支持 Bearer token 认证下载
- 统一 `downloadSingleFile` 消除代码重复
- 静默 C 库调试输出，命令行输出更干净
