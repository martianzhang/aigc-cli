# v2.0.0 — 命名 Provider 架构

> 各命令可以独立配置不同厂商，离线命令支持在线 LLM 后端。

## 🏗️ 命名 Provider 架构

**配置方式**：在 `providers` 段定义可复用的命名 Provider，各命令通过 `defaults.{cmd}.provider` 引用：

```yaml
providers:
  my-openrouter:
    type: openai
    api_key: "sk-or-xxx"
    base_url: "https://openrouter.ai/api/v1"
  my-ollama:
    type: ollama
    base_url: "http://localhost:11434/v1"

defaults:
  image:
    provider: "my-openrouter"
    model: "openai/gpt-image-2"
  ocr:
    provider: "my-ollama"
    model: "glm-ocr"
```

**四层优先级**：`--api-key/--api-base` > `--provider` > `defaults.{cmd}.provider` > 全局 `api_key/base_url`

**Provider type**：支持 `openai`（默认）、`ollama`（本地，免 API Key）、`google`/`anthropic`（预留）

## 🚀 新功能

| 功能 | 说明 |
|---|---|
| `--provider` 全局标志 | 临时切换厂商，`--provider local` 走本地 ONNX |
| `ocr scan --prompt` / `--ask` | 自定义 OCR 提示词，支持模型特定格式 |
| `vision describe --prompt` / `--ask` | 自定义图片描述提示词 |
| `--print-config` 展示 provider | 各命令有效 Provider 一目了然 |
| OCR 对接 Ollama | 支持 deepseek-ocr、glm-ocr |
| Vision 对接 Ollama | 支持 gemma4-vision 等 |

## ⚠️ 不兼容变更

| 变更 | 旧行为 | 新行为 |
|---|---|---|
| `AudioDefaults.Local` 字段 | `config.yaml` 中 `audio.local: true` 控制本地模式 | 移除，改为 `--provider local` 或 `provider: local` |
| `ocr/vision` 默认行为 | 始终走本地 ONNX | 无 provider 时走本地，配了 provider 走在线 |
| `isAPIMartProvider()` / `isOpenRouterProvider()` | 公开函数 | 标记 deprecated，改用 `p.ProviderType` |
