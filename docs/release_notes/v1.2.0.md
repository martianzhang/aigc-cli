# v1.2.0 — 百度 & 智谱清言 水印引擎 + 自动路由

在 v1.1.0 水印引擎基础上扩展两个新平台，并大幅提升 TC260 元数据到水印配置的自动路由能力。

## 新功能

- **百度文心一言去水印** — "百度 AI生成" 嵌入式文字水印（187×51 @1024px native），右下角，走 `RemoveAlphaBlend` 通道
- **智谱清言（Zhipu / ChatGLM）去水印** — "智谱清言" 嵌入式文字水印（234×60 @1024px native），右下角，通过两拍法（`visible_alpha_solve.py`）从纯黑/纯灰种子图求解精确 alpha map
- **TC260 → 水印配置品牌别名自动路由** — TC260 元数据里的中文/英文厂商描述（如 `字节跳动 (ByteDance) — 豆包`、`智谱AI (Zhipu) — 清言 / GLM`、`Google DeepMind`、`Baidu`）通过 `brandAliases` 子串匹配自动映射到注册的 watermark 配置，无需 `--producer` 也能命中正确引擎
- **`--producer zhipu`** — `detect` 命令新增 zhipu 选项（`service.ProviderZhipu`）

## 改进

- **豆包去水印质量恢复** — 回退到 v1.0 之前的核心去除路径（保留 `w/h` 矩形传参、逆 alpha 数学、`meanBackgroundLuma` 亮背景守卫、oversubtract 惰性守卫），恢复 4 张 `.testdata/doubao` 图像的 pixel-identical 去除效果（MSE=0.00，PSNR=99.9 dB）
- **Oversubtract margin 数据驱动** — `Config.OversubtractMargin` 取代硬编码阈值（默认 25），便于按厂商调参
- **`doubao-snap` alpha map 尺寸修正** — 从错误的 774×327 缩放回正确的 118×58，与 badge 实际 CSS 渲染尺寸一致

## Bug 修复

- 修复 lint 告警：删除 `watermark.findConfig`（无调用者）+ `gemini.resolveWatermarkConfigs` 内层 `dimKey` 局部类型重影；将 `cmd/detect.go` 两处 De Morgan 反向条件改写为等价正向表达

## 新脚本 & 辅助工具

- `scripts/verify_watermark.py` — `--producer` 选项新增 `baidu` / `zhipu`，`WM_PARAMS` 表添加两平台几何参数
- `scripts/check_watermark.py` — `WATERMARK_CONFIGS` 新增 `zhipu` 诊断条目
